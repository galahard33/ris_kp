# GaussWebApp — Веб-интерфейс для решения СЛАУ методом Гаусса

Это веб-приложение на **ASP.NET Core MVC (C#)** позволяет:

- 🧮 **Генерировать** случайные невырожденные матрицы заданного размера (с диагональным преобладанием),
- 💾 **Скачивать** сгенерированную матрицу в виде текстового файла,
- 📤 **Загружать** собственные матрицы,
- ⚡ **Решать СЛАУ** методом Гаусса в **1, 2, 4 или 8 потоков**,
- 📊 **Сравнивать производительность** и точность решений,
- 👁️ **Просматривать** фрагменты матрицы и решения в удобном интерфейсе.

## 🚀 Быстрый запуск

### Запуск с Docker Compose (рекомендуемый способ)

1. **Клонируйте репозиторий** :
   
   -`git clone https://github.com/Galahard33/ris_kp.git`

   -`cd ris_kp`
2. **Запустите приложение одной командой**: 
   
   -`docker compose up --build`
   эта команда запустит веб интерфейс и 4 воркера для распределительных вычислений
3. **Приложение будет доступно** по адресу:
   - 🌐 **Веб-интерфейс**: [http://localhost:5052](http://localhost:5052)

## 🧪 Тестирование

### Вариант 1: Запуск всех тестов через скрипт (Unix/Linux/macOS)

Дайте права на выполнение скрипта - `chmod +x ./run-tests.sh`

Запустите все тесты - 
`./run-tests.sh`

Скрипт автоматически выполнит:
- ✅ **Юнит-тесты** (проверка корректности работы)
- 📊 **Нагрузочные тесты** (тестирование производительности)
- ⏱️ **Измерение времени** выполнения каждого теста
- 📋 **Вывод результатов** в консоль

### Вариант 2: Запуск тестов по отдельности

#### Юнит-тесты (тестирование функциональности):
- `cd Tests/GaussWebApp.UnitTests/`
- `dotnet test`

#### Нагрузочные тесты (тестирование производительности):
- `cd Tests/GaussWebApp.LoadTests/`
- `dotnet test`

### Вариант 3: Запуск тестов через Docker

Запуск всех тестов в контейнерах:

- `docker compose -f 'docker-compose.test.yml' up --build`

## 📈 Визуализация результатов тестирования

После выполнения нагрузочных тестов в папке `Tests/GaussWebApp.LoadTests/` появятся CSV-файлы с результатами:

- `scalability-results.csv` — результаты масштабируемости (время, ускорение, эффективность)
- `memory-usage.csv` — использование памяти для разных размеров матриц

### Генерация графиков результатов

1. **Установите зависимости Python**:
- `cd Tests/GaussWebApp.LoadTests/`
- `pip install -r requirements.txt`

2. **Запустите скрипт визуализации**:

    **Для Unix/Linux/macOS**:

    - `python3 visualize_results.py`

    **Для Windows**:
    - `python visualize_results.py`

3. **После выполнения будут созданы**:
- `scalability_analysis.png` — графики масштабируемости
- `memory_analysis.png` — графики использования памяти

### Параметры визуализации

Скрипт `visualize_results.py` автоматически:
- Строит графики зависимости времени выполнения от количества потоков
- Анализирует ускорение и эффективность параллелизации
- Оценивает сложность алгоритма (O(n³))
- Анализирует использование памяти для разных размеров матриц
